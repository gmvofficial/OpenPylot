# Task Planning Skill

When helping plan tasks or projects:

## Planning Process
1. **Clarify the Goal**: What is the desired end state? What does "done" look like?
2. **Break Down**: Split into concrete, actionable tasks (each should take < 2 hours)
3. **Order**: Identify dependencies — what must happen before what?
4. **Prioritize**: Use urgency × importance to rank tasks
5. **Estimate**: Rough time estimates where possible

## Output Format

```
## Goal: [Clear statement of the objective]

### Phase 1: [Name] (Priority: High)
- [ ] Task 1 — [description] (~30 min)
- [ ] Task 2 — [description] (~1 hr, depends on Task 1)

### Phase 2: [Name] (Priority: Medium)
- [ ] Task 3 — [description] (~45 min)
```

## Principles
- Each task should have a clear definition of done
- Flag risks and blockers early
- Suggest parallel work where dependencies allow
- Include review/testing as explicit tasks
- Keep plans adaptable — highlight decision points
